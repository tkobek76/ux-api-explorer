Benefits Domain APIs

Summary

The Benefits domain includes information related to current and future employee benefit enrollment. This includes medical enrollment information, flexible spending contribution amounts, retirement contributions and more.

Benefits

Benefit Management

Participant Benefit Elections is the list of elections made by a participant for organization provided benefits. This specification details the Election Summary which is a list of health and welfare benefits Elections which the participant has (in force and future), and the Election Detail which gives the details of a particular benefit election.

Available APIs: Benefits > Benefit Management

# API Name Description Action
1 Benefit Elections v1 Request the list of benefit coverage which is effective for the participant (current year). View API
2 Benefit Election v1 Request the details of a benefit coverage. View API
Benefits

Flexible Spending Account (FSA) Management

This API supports Flexible Spending Accounts. It returns the list of accounts, corresponding balances, transactions against the account (e.g. claims, disbursements), and any cards which the user has associated to the account. Financial Accounts are accounts which hold monies of an account holder and have transactions which move monies in and out of the account. An example of a financial account is a Flexible Spending Account, a Commuter Account. These APIs support management of data against a financial account such as balances, transactions, card holders, money movement, etc…

Available APIs: Benefits > Flexible Spending Account (FSA) Management

# API Name Description Action
1 Flexible Spending Account v1 Request the financial accounts of a particular type. View API
2 Flexible Spending Account Transaction v1 Request a set of account transactions View API
Benefits

Retirement Plan

Retirement Plans communicates the list of retirement plans that an individual has based on an election or a core benefit provided by the employer. The list provides references to the details of each retirement plan. Each retirement plan provides details of balance (vested/unvested) as of a valuation date as well as investment allocations and loan details.

Available APIs: Benefits > Retirement Plan

# API Name Description Action
1 Retirement Plan Election v1 Request the list of retirement plans for an Associate with an Organization. Examples are 401k and Roth – IRA View API
2 Retirement Plan Details v1 Request the details of the retirement plan View API

Worker

The Workers API supports the management of workers. A worker is a person who performs duties and responsibilities for an organization. In that capacity, a worker may fill one or more positions via a work assignment. For a given work assignment, a worker is related to the organization as

  • An employee or permanent worker
  • A contractor or fixed term worker, e.g. seasonal workers, interns
  • A casual worker
  • An apprentice
  • A vendor employee

Benefits

Retirement Plan

Retirement Plans communicates the list of retirement plans that an individual has based on an election or a core benefit provided by the employer. The list provides references to the details of each retirement plan. Each retirement plan provides details of balance (vested/unvested) as of a valuation date as well as investment allocations and loan details.

Available APIs: Benefits > Retirement Plan

# API Name Description Action
1 Retirement Plan Election v1 Request the list of retirement plans for an Associate with an Organization. Examples are 401k and Roth – IRA View API
2 Retirement Plan Details v1 Request the details of the retirement plan View API

Benefit Management

Participant Benefit Elections is the list of elections made by a participant for organization provided benefits. Flexible Spending Account (FSA) Management

This API supports Flexible Spending Accounts. It returns the list of accounts, corresponding balances, transactions against the account (e.g. claims, disbursements), and any cards which the user has associated to the account. Financial Accounts are accounts which hold monies of an account holder and have transactions which move monies in and out of the account. An example of a financial account is a Flexible Spending Account, a Commuter Account. These APIs support management of data against a financial account such as balances, transactions, card holders, money movement, etc…
